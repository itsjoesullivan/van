schroeder

Schroeder is a hardware music keyboard built on a web stack.

The stack

Under the cover is a Chromebox running an app I developed 'specially for schroeder

The other hardware is a midi keyboard ( a Korg microKEY 25 ), an amp and speaker, and a wooden case.

The app

The app is a Chrome packaged app. It uses the USB API to listen to the keyboard (here's a gist to the code for that), maps notes/velocity to audio files, and uses the Web Audio API to play them.

The sound

The sound is the best Wurlitzer sound I could practically get.

I'd have loved to rent and sample one myself, but there was _some_ limit to the scope of this project, so I settled on the second best thing: sampling Native Instruments' Scarbee A-200

And, frankly speaking, I'm sure that sampling THEIR samples turned out better than recording them myself.

For the actual sampling:

  1. Load up the keyboard in GarageBand
  2. Create a midi region containing the notes you want to play. For me, it was four octaves sampled at the C and F# with five velocity layers
  3. Bounce down the project into a single wav
  4. Use a bash script and SoX to split up the file, using a helpful naming convention
  5. Request the files on the client side
